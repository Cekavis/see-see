# Quickstart & Validation Guide

本文件描述实现完成后的本地启动和验收路径；当前 `/speckit-plan` 阶段不会创建应用代码或安装依赖。

## Prerequisites

- Windows 10/11，或 macOS 14+
- Rust 1.95 toolchain
- Node.js 24 LTS 与 npm
- macOS：允许终端/开发构建请求“屏幕录制”权限
- 一个可接受图片输入的 OpenAI、Anthropic、Gemini 或兼容端点

API Key 只在应用设置界面输入。不要把 Key 写入 `.env`、命令行、测试输出或仓库文件。

## Install and run

```powershell
npm install
npm run tauri dev
```

预期结果：首次启动显示简体中文引导；关闭普通窗口后应用仍留在系统托盘或 macOS 菜单栏。

## Automated checks

```powershell
npm run typecheck
npm run lint
npm test
npm run test:e2e
cargo fmt --manifest-path src-tauri/Cargo.toml --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings
cargo test --manifest-path src-tauri/Cargo.toml
npm run tauri build
```

最小必需覆盖：

- URL 规则：远程 HTTP 被拒绝，本机回环 HTTP 被接受，URL 中凭据被拒绝。
- 三类 provider：图片请求 JSON、认证 header、模型列表、流式文本增量、空结果和错误归一化。
- 分析状态机：单活动请求、取消不落历史、失败不自动重试、终止事件唯一。
- 数据：成功记录两表同事务写入、删除级联、关闭历史不落库、10,000 条参数化搜索/筛选基准。
- 秘密：IPC 列表、数据库、普通日志和导出日志中不存在测试 API Key。
- UI：加载/空白/成功/失败/取消/禁用/恢复状态，键盘焦点，复制、删除确认和历史再次提交。

## Manual primary flow

1. 启动应用，确认开机启动默认未勾选。
2. 在 macOS 授予屏幕录制权限；在 Windows 确认无需额外权限即可进入下一步。
3. 新建模型配置，填写协议、端点、模型和 API Key。
4. 执行连接测试，确认界面说明测试会发送极小图片请求，并显示成功或分类错误。
5. 选择“日语学习解析”提示词。
6. 关闭设置窗口，使用全局快捷键框选一张包含日文的图片。
7. 确认 500 ms 内出现选区，松开后 200 ms 内出现提交状态，模型返回时逐步显示文本。
8. 验证结果浮窗默认置顶、可取消置顶、可调整大小、`Esc` 可关闭、“复制全文”可用。
9. 打开历史，确认原图、缩略图、提示词快照、服务配置名、模型、时间和完整结果存在。
10. 搜索结果关键词、按提示词筛选，再次提交该截图；确认新请求使用当前模型和提示词并产生新记录。
11. 删除单条记录和全部清空，确认需要明确确认且列表立即更新。

## Privacy and failure validation

1. 关闭“保存历史”，完成一次截图分析并关闭结果浮窗；重启后确认无该截图和输出。
2. 输入错误 API Key，确认返回认证错误且不自动重试；检查日志不存在 Key 和完整响应体。
3. 配置远程 `http://` 地址，确认保存被拒绝；配置 `http://localhost`，确认允许继续测试。
4. 在请求中断网，确认显示网络错误、当前结果不被标记成功，并可由用户手动重试。
5. 生成过程中再次按快捷键，确认聚焦现有结果而不是产生第二次请求。
6. 取消生成，确认不写历史且内存中的截图和部分结果被释放。

## Cross-display validation

Windows 与 macOS 各执行：

- 单显示器 100% 缩放。
- 高像素密度显示器。
- 两个显示器左右排列，分辨率不同。
- Windows 两个显示器缩放比例不同，且次显示器位于主显示器左侧形成负坐标。
- 选区完全位于单屏、跨越两屏、从右下向左上拖动、零尺寸取消。

所有情况下，截图结果必须与可见选区一致，不包含 overlay，文字不得因坐标换算被截断。

## Accessibility and visual review

在 Windows 与 macOS 各保存以下证据截图：

- 1024×720 与 1440×900 的首次引导、设置和历史界面。
- 360×240 的最小结果浮窗。
- 历史空态、搜索无结果、成功详情、失败详情。
- 模型配置加载、测试成功、认证失败和 TLS 失败。
- 键盘焦点依次经过主要操作；200% 文字缩放下无关键控件不可达。

人工确认视觉层级、间距、文字截断、对比度、焦点可见性和平台惯例一致性，并把证据路径记录到实现任务的验证说明中。

## Accuracy benchmark

准备 100 张不包含个人敏感信息的日文截图，覆盖小字号、竖排、混合排版、低对比度和带背景图片文本。固定同一提示词、同一指定多模态服务和一个固定 OCR+翻译基线，由双语评审按统一量表判断原文与译意是否可用。

通过标准：多模态流程可用率至少 85%，且比 OCR 基线高至少 10 个百分点。基准数据、模型标识、提示词快照和评审结果必须随发布验证记录保存，不进入产品遥测。
