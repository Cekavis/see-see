# Feature Specification: See See 多模态截图翻译

**Feature Branch**: 未创建（未配置分支创建钩子）

**Created**: 2026-07-23

**Status**: Draft

**Input**: 创建一个 Windows 与 macOS 截图翻译软件，跳过 OCR，直接将截图交给用户配置的多模态模型；支持主流 API 协议、自定义端点、自定义提示词、可配置快捷键和完善的本地历史记录。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 快捷截图并获得模型解析结果 (Priority: P1)

用户按下全局快捷键框选屏幕区域，松开鼠标后，See See 直接把截图和当前提示词发送给当前多模态模型配置，并在独立浮窗中展示持续生成的结果。整个流程不经过 OCR，也不要求用户先打开主窗口。

**优先级理由**：这是产品区别于传统 OCR 翻译工具的核心价值，单独完成即可交付可用的截图理解流程。

**独立测试**：配置一个可用的图像理解模型和提示词后，从任意应用框选一段日文，验证结果浮窗展示原文、中文翻译、词语解析和语法说明。

**验收场景**：

1. **假如**应用已在后台运行且当前模型配置有效，**当**用户按快捷键并完成区域框选，**那么**应用立即提交截图和当前提示词，并显示明确的处理中状态。
2. **假如**端点支持流式响应，**当**模型开始返回内容，**那么**浮窗逐步展示已生成文本，用户无需等待完整回答。
3. **假如**端点不支持流式响应，**当**请求仍在处理，**那么**浮窗持续显示处理中状态，并在完整结果到达后一次性展示。
4. **假如**用户正在框选，**当**用户取消截图，**那么**应用不发送请求、不创建历史记录，并恢复到截图前状态。
5. **假如**一个请求正在生成，**当**用户再次触发截图快捷键，**那么**应用聚焦当前结果浮窗并提示用户完成或取消现有请求。
6. **假如**结果已显示，**当**用户选择文字或点击“复制全文”，**那么**对应内容进入系统剪贴板；关闭浮窗后，已保存的结果仍可从历史记录打开。

### User Story 2 - 配置自己的多模态模型服务 (Priority: P1)

用户可以保存多个命名的模型配置，从预设端点或自定义端点中选择服务，并指定协议、模型和凭据。应用不提供账号、模型额度或请求中转服务。

**优先级理由**：没有可用的用户自备模型配置，核心截图流程无法工作；多协议与自定义端点也是产品覆盖不同服务商的基础。

**独立测试**：分别创建三类预设协议配置和一个自定义端点配置，完成连接测试、切换当前配置并用截图得到结果。

**验收场景**：

1. **假如**用户首次启动且没有有效模型配置，**当**用户触发截图，**那么**应用引导用户完成配置，而不是发送必然失败的请求。
2. **假如**用户创建模型配置，**当**端点能够提供模型列表，**那么**用户可以从列表选择模型，同时仍可手动填写模型标识。
3. **假如**端点不能提供模型列表，**当**用户手动填写模型标识并执行连接测试，**那么**应用仍可验证端点、凭据和图像输入能力。
4. **假如**用户填写远程明文 HTTP 地址，**当**保存配置，**那么**应用拒绝保存并说明必须使用 HTTPS；本机回环地址不受此限制。
5. **假如**配置连接测试失败，**当**应用获得失败原因，**那么**界面区分网络、认证、限流、服务异常和模型不支持图片等可操作错误。
6. **假如**用户切换当前模型配置，**当**之后执行截图，**那么**新请求使用新配置，已有历史记录仍保留原调用信息。

### User Story 3 - 创建和切换提示词预设 (Priority: P2)

用户可以管理多个命名提示词预设，并选择一个当前预设决定截图的分析目标与输出格式。语言和输出结构由提示词表达，不使用独立的源语言或目标语言选择器。

**优先级理由**：自定义提示词让产品从固定翻译器扩展为面向学习、排版理解和其他图像文本任务的工具。

**独立测试**：分别使用内置“通用翻译为中文”和“日语学习解析”预设处理同一截图，验证两次输出按不同要求生成。

**验收场景**：

1. **假如**用户尚未创建提示词，**当**首次完成引导，**那么**应用提供“通用翻译为中文”和“日语学习解析”两个可直接使用的预设。
2. **假如**用户管理提示词，**当**新增、编辑、复制、删除或切换预设，**那么**列表与当前预设状态立即保持一致。
3. **假如**当前没有可用提示词，**当**用户触发截图，**那么**应用阻止提交并引导用户选择或创建提示词。
4. **假如**用户从历史记录再次提交截图，**当**当前提示词与原记录不同，**那么**新请求使用当前提示词，并生成独立的新历史记录。

### User Story 4 - 查看和管理完整本地历史 (Priority: P2)

用户可以在历史界面查看过去的截图、生成结果和调用信息，搜索或筛选记录，重新打开、复制、再次提交或删除记录。历史默认长期保存在本机，但用户可以关闭历史记录功能。

**优先级理由**：长结果经常需要回看、比较和复用；历史记录是用户明确要求的首版核心界面。

**独立测试**：连续完成多次截图后，通过关键词和提示词筛选找到目标记录，打开原图与结果，复制内容、再次提交并删除记录。

**验收场景**：

1. **假如**历史记录已开启，**当**一次请求成功完成，**那么**应用保存原始截图、缩略图、完整结果、时间、提示词快照、服务类型、模型和状态。
2. **假如**历史记录已开启且请求失败，**当**用户打开历史界面，**那么**失败记录显示时间、调用信息和脱敏错误摘要，但不伪造空的成功结果。
3. **假如**用户取消尚未完成的请求，**当**返回历史界面，**那么**历史中不存在该次不完整请求。
4. **假如**历史中存在大量记录，**当**用户输入结果文本关键词或按提示词筛选，**那么**列表只显示匹配记录，并保留明确的空结果状态。
5. **假如**用户点击“再次提交”，**当**当前模型配置和提示词有效，**那么**应用用原截图和当前配置创建新请求，原记录不被覆盖。
6. **假如**用户关闭“保存历史”，**当**后续请求完成，**那么**截图和结果只保留在当前浮窗中，关闭浮窗后清除。
7. **假如**用户删除单条记录或选择全部清空，**当**确认操作，**那么**关联截图和结果从本地历史中删除，界面立即反映变化。

### User Story 5 - 在桌面后台可靠运行 (Priority: P3)

用户可以通过 Windows 系统托盘或 macOS 菜单栏访问截图、历史记录、提示词、服务设置和退出功能，并可配置全局快捷键及是否随系统启动。

**优先级理由**：后台运行、权限和快捷键管理让核心流程在日常使用中可被稳定触发，但不改变基本的模型分析能力。

**独立测试**：在 Windows 和 macOS 上完成首次引导，关闭所有窗口后通过托盘或菜单栏和快捷键继续使用截图功能。

**验收场景**：

1. **假如**用户首次启动，**当**进入引导，**那么**应用依次处理截图权限、模型配置和连接测试，并提供默认提示词与快捷键。
2. **假如**macOS 尚未授予屏幕录制权限，**当**用户尝试截图，**那么**应用解释所需权限、提供前往系统设置的入口，并在授权前保持可恢复状态。
3. **假如**用户设置的快捷键与现有快捷键冲突，**当**保存设置，**那么**应用拒绝启用冲突组合并要求重新选择。
4. **假如**用户关闭设置、历史或结果窗口，**当**应用仍在后台运行，**那么**快捷键继续可用；只有选择“退出”才完全结束应用。
5. **假如**首次引导询问开机启动，**当**用户未主动勾选，**那么**应用不随系统自动启动。

### Edge Cases

- 截图区跨越多个显示器、不同缩放比例或高像素密度时，选区和生成的截图必须保持一致，不得错位或裁切错误。
- 用户按下快捷键后未形成有效选区、选区尺寸为零或截图失败时，不得发送请求或创建历史记录。
- 截图尺寸或数据量超过端点限制时，应用必须给出可理解的错误与恢复建议，不得无限等待或静默丢弃内容。
- 当前模型配置或提示词在请求开始前被删除、失效或切换时，应用必须使用请求启动时确定的有效配置；无法确定时阻止提交。
- 请求期间断网、系统睡眠、端点超时或应用退出时，应用必须终止未完成状态并避免保存为成功记录。
- 模型返回空内容、非文本内容或异常格式时，应用必须显示可诊断错误，不得将其呈现为正常翻译。
- 历史原图缺失或损坏时，用户仍可查看其余元数据和已保存结果，但不能再次提交，并应看到明确提示。
- 删除当前使用中的模型配置或提示词后，应用必须清除对应当前选择，并在下一次截图前要求重新选择。
- 系统剪贴板不可用或复制失败时，应用必须保留结果并提示用户重试。
- 本地存储空间不足时，应用必须停止新增历史数据、保留当前可见结果并提示用户清理历史。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：系统必须在 Windows 10/11 与受支持的 macOS 版本上提供区域截图能力。
- **FR-002**：系统必须允许用户配置唯一的全局“区域截图并提交”快捷键，并检测不可用或冲突的组合。
- **FR-003**：系统必须在用户完成有效框选后直接把原始截图交给当前多模态模型，不得执行本地 OCR、远程 OCR 或 OCR 失败回退。
- **FR-004**：系统必须将当前提示词与截图作为同一次模型请求的输入。
- **FR-005**：系统必须提供可取消的处理中状态，并在端点支持时持续展示流式文本。
- **FR-006**：系统必须限制同一时间最多存在一个生成中的请求，并在重复触发快捷键时聚焦现有请求。
- **FR-007**：系统必须提供独立、可调整大小的结果浮窗；浮窗默认置顶且可取消置顶。
- **FR-008**：用户必须能够选择并复制结果文本、复制全文，并使用 `Esc` 关闭结果浮窗。
- **FR-009**：系统必须支持 OpenAI、Anthropic 和 Gemini 三类模型服务协议。
- **FR-010**：系统必须提供上述协议的预设端点，并允许用户为所选协议填写自定义端点。
- **FR-011**：系统必须允许用户保存、编辑、删除、切换多个命名模型配置，并指定协议、端点、模型标识和所需凭据。
- **FR-012**：系统必须在端点支持时展示可选模型列表，同时始终允许手动填写模型标识。
- **FR-013**：系统必须提供连接测试，验证端点可达、凭据有效、所选模型可接受图片并返回文本。
- **FR-014**：系统必须拒绝非本机地址的明文 HTTP 端点；`localhost` 和回环地址可以使用 HTTP。
- **FR-015**：系统不得提供忽略无效 TLS 证书的选项。
- **FR-016**：系统的网络请求必须遵循操作系统代理设置，首版不提供独立代理配置。
- **FR-017**：系统必须把 API Key 等秘密凭据保存到操作系统提供的安全凭据存储中，不得明文写入普通配置、历史或日志。
- **FR-018**：系统不得提供内置账号、模型额度、计费或远程请求中转服务；请求从用户设备发送到用户选择的端点。
- **FR-019**：系统必须允许用户新增、编辑、复制、删除和切换多个命名提示词预设。
- **FR-020**：系统必须内置“通用翻译为中文”和“日语学习解析”两个提示词预设。
- **FR-021**：提示词必须以单段普通文本管理；首版不得提供变量、条件逻辑、角色分栏或独立源语言/目标语言选择器。
- **FR-022**：系统必须默认启用本地历史记录，并允许用户关闭保存历史。
- **FR-023**：历史记录已开启时，成功历史记录必须保存原始截图、缩略图、完整输出、创建时间、提示词文本快照、服务协议、模型标识和状态。
- **FR-024**：历史记录已开启时，失败历史记录必须保存脱敏错误摘要和调用元数据，不得保存为空的成功结果；用户主动取消的请求不得进入历史。
- **FR-025**：历史界面必须支持浏览、结果文本搜索、按提示词筛选、重新查看、复制、再次提交、单条删除和全部清空。
- **FR-026**：历史记录必须默认长期保留，直到用户单独删除、全部清空或关闭历史后不再新增。
- **FR-027**：系统必须允许用历史原图和当前提示词、当前模型配置再次提交，并将结果保存为新记录，不得覆盖原记录。
- **FR-028**：本地历史首版不提供云同步、标签、文件夹或应用级加密，并必须向用户明确说明截图保存在本机。
- **FR-029**：系统必须作为后台桌面应用运行，并通过 Windows 系统托盘或 macOS 菜单栏提供截图、历史、提示词、模型配置和退出入口。
- **FR-030**：关闭普通窗口不得退出后台应用；用户必须通过明确的“退出”操作结束应用。
- **FR-031**：系统必须提供可选的开机自动启动设置，首次引导中默认不启用。
- **FR-032**：首次启动必须引导用户完成必要截图权限、模型配置和连接测试；没有有效模型配置或提示词时不得提交截图。
- **FR-033**：系统必须对网络、认证、限流、端点异常、模型不支持图片和本地存储失败提供不同的错误说明和恢复操作。
- **FR-034**：系统不得自动重试失败请求；只能由用户主动重试，以避免重复计费。
- **FR-035**：系统不得收集或上传使用分析、截图内容、提示词、模型输出或默认崩溃报告。
- **FR-036**：系统可以保留本地诊断日志，但日志必须排除 API Key、截图内容和完整模型输出，并支持用户主动导出脱敏日志。
- **FR-037**：系统界面首版必须使用简体中文，所有用户可见文本必须可由语言资源替换；首版不提供语言切换器。
- **FR-038**：首版输入必须仅限应用内区域截图，不得提供文件导入、拖放图片或剪贴板图片入口。
- **FR-039**：首版不得提供截图编辑或标注、多轮聊天、并发请求队列、每个提示词独立快捷键或高级生成参数。
- **FR-040**：系统必须在保存历史前确认截图和输出都已完整写入；任一部分失败时不得产生表面完整但内容缺失的成功记录。

### User Experience and UI Requirements *(mandatory for user-facing features)*

- **UX-001**：截图、模型配置、提示词、结果浮窗和历史界面必须分别定义加载、空白、成功、错误、禁用和恢复状态；相同状态使用一致的文字、颜色和操作位置。
- **UX-002**：Windows 使用用户熟悉的系统托盘、快捷键和窗口行为；macOS 使用菜单栏、系统权限引导和平台对应快捷键标识。跨平台核心流程和功能名称必须一致。
- **UX-003**：截图开始、请求提交、流式生成、完成、取消和失败必须提供即时且不含歧义的反馈；不得用静默等待表示网络工作。
- **UX-004**：删除历史、全部清空和删除当前配置等不可逆操作必须明确目标与影响，并要求确认；普通取消和关闭操作不得增加确认阻力。
- **UX-005**：所有主要操作必须支持键盘导航，焦点顺序可预测，焦点状态清晰；常用操作必须具有可读标签，不能只依赖图标、颜色或悬停提示。
- **UX-006**：文字与交互控件必须达到通用桌面无障碍对比度和可点击尺寸要求，并兼容系统文字缩放与减少动态效果设置。
- **UX-007**：结果内容必须作为可选择文本呈现；模型输出可以按安全的富文本格式展示，但不得执行其中的脚本、远程内容或交互式标记。
- **UX-008**：历史主界面在 1024×720 与 1440×900 的代表性窗口尺寸下必须保持搜索、筛选、列表和详情可用；结果浮窗在 360×240 的最小尺寸下仍可访问主要操作。
- **UX-009**：设置与历史窗口调整大小后不得遮挡关键操作或丢失内容；小窗口下允许滚动，但不得产生无法访问的固定区域。
- **UX-010**：发布前必须在 Windows 与 macOS 各完成一次人工视觉审查，记录截图框选、首次引导、结果浮窗、历史空态/列表/详情、错误态和键盘焦点的证据。

### Key Entities

- **模型配置**：一个用户命名的服务连接，包含协议类型、端点、模型标识、非秘密设置、安全凭据引用、连接测试状态和是否为当前配置。
- **提示词预设**：一个用户命名的普通文本指令，包含名称、正文、是否为当前预设和创建、修改时间。
- **分析请求**：一次截图到模型结果的运行过程，包含请求开始时固定的截图、提示词和模型配置、运行状态、时间和脱敏错误信息。
- **历史记录**：已完成或失败分析请求的本地持久化视图，包含原图、缩略图、完整结果、提示词快照、调用元数据、状态和创建时间。
- **应用设置**：跨会话保存的非秘密偏好，包括截图快捷键、是否保存历史、是否开机启动、窗口偏好和当前模型、提示词选择。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：至少 90% 的首次使用测试者能在 5 分钟内完成权限、模型连接和第一次成功截图分析，无需外部指导。
- **SC-002**：在代表性 Windows 与 macOS 设备上，用户触发快捷键后 500 毫秒内看到可交互的截图选区；完成框选后 200 毫秒内看到已提交或错误反馈。
- **SC-003**：在指定的图像理解服务与 100 张包含小字号、竖排、混合排版和低对比度文本的代表性截图上，双语评审判定“原文识别与译意均可用”的比例达到 85%，并比固定 OCR 后翻译基线高至少 10 个百分点。
- **SC-004**：至少 95% 的测试者能在一次尝试内完成“截图、等待结果、复制全文”的主任务。
- **SC-005**：包含 10,000 条本地记录时，95% 的历史打开、关键词搜索和提示词筛选操作在 2 秒内呈现完整可操作结果。
- **SC-006**：Windows 与 macOS 的核心验收场景全部通过，且不存在仅在单个平台可完成的 P1 或 P2 用户故事。
- **SC-007**：安全验证中，普通配置、历史、诊断日志和导出日志均不出现明文 API Key；关闭历史后完成的请求在关闭结果浮窗后不留下可恢复的截图或输出记录。
- **SC-008**：无障碍与视觉审查覆盖所有规定界面和状态，键盘可达率达到 100%，不存在阻断核心任务的对比度、焦点或文字缩放问题。
- **SC-009**：网络、认证、限流、图片能力和存储失败测试中，100% 的错误都显示与原因对应的恢复建议，且不会自动产生第二次计费请求。

## Assumptions

- 目标用户拥有至少一个可接受图片输入的兼容模型服务账号或可访问的本地模型端点，并自行承担模型使用费用。
- 用户设备能够联网访问所选远程服务；本地端点可以不要求 API Key，但仍必须通过连接测试。
- “受支持的 macOS”指产品发布时仍获得安全更新、且能够授予屏幕录制权限的版本；具体最低版本在实施规划阶段根据所选桌面能力确定。
- 首版只验收 Windows 10/11 和 macOS，不承诺 Linux、移动端或浏览器版本。
- 模型输出准确度取决于用户选择的模型、提示词和截图质量；产品负责忠实提交原图、提供可验证的参考基准和清楚展示结果，不承诺所有第三方模型达到同等准确度。
- 历史数据属于当前操作系统用户，依赖操作系统账户与应用数据目录权限隔离；共享设备用户应关闭历史或主动清空记录。
- 预设端点和协议兼容范围不意味着维护固定模型目录；模型标识允许用户手动输入，以适应服务商更新。
